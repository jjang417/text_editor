[How to submit change to mainline] 

 

Generally the best process is to merge changes into the mainline tree, then port them to other trees using p4 integrate as much as possible. If your CL is working on a particular tree we can start with that one first. 

  

In whatever tree your changes are, you should run the CSIM sanity command and make sure the tests are passing: 

gsim -- tgen_rwmp.pl -arch Linux_x86_64 -target RS_CSIM -chip tb500 --testArgsMore -force_linux_64bit -failOnZeroTests -level as2/tb500/as2_sc_factory_sanity_64.lvl -maxFileSize 1000000 -queue o_cpu_16G_1H 

  

Then it's a good idea to ask people to review your changes once you know they are working. 

  

In the nvmobile_tb50x_cmod_NET_Rel_<x> trees, you would only then have to use as2 submit to sumbit your changes. 

  

In the mainline tree there are other things that must be done first. 

  

In particular, you'll need 2 upvotes on your review in the mainline tree. 

  

Also, you'll need to run the as2 presubmit command like this first: 

nvrun scsim_as2_pre_checkin_sanity.sh tb500 

  

And then paste the resulting nvregress link from that regression into your CL description before as2 submit'ting. 

 

as2 submit –c <HW CL>
