

  Step 1 — Kick it off

  Download now? [y/N]

  Prompt #2 (in phase 5, ~3 min in):                                                                                                 
  Credentials baked into the install:                                                                                                
    username: nvidia                                                                                                                 
    password: nvidia   (hashed with SHA-512 in seed)                                                                                 
    hostname: l1-ubuntu                             
  Hypervisor stack (qemu-kvm, libvirt) will be preinstalled inside the L1.

  → press y.                                                                                                                         
                                                                                                                                     
  Step 3 — Watch the install (~10–15 min)                                                                                            
                                                                                                                                     
  Install logs will scroll by on serial console. You'll see kernel boot, then subiquity (Ubuntu's installer) doing partitioning,     
  package install, etc. Don't touch anything. It auto-powers-off when done.
                                                                                                                                     
  If it stalls for >5 min on the same line, that's likely a problem — let me know what you see.

  Started 'l1-ubuntu'.                                                                                                               
                                                                                                                                     
  ===== All set =====                                                                                                                
    Login as: nvidia / nvidia                                                                                                        
    ...                                                                                                                              
                                                                                                                                     
  Step 5 — Log in to verify
                                                                                                                                     
  sudo virsh console l1-ubuntu                              
  # wait for the login prompt (may take 30-60s for L1 to finish booting)                                                             
  # log in: nvidia / nvidia                                                                                                          
  # ctrl-] to exit when done 

  Or via SSH:                                                                                                                        
    sudo virsh net-dhcp-leases default     # find L1's IP
    ssh nvidia@192.168.122.<x>                                                                                                           
                                                                                                                                     
  Inside the L1, sanity-check:                                                                                                       
  free -h                        # should show ~256 GiB total
  nproc                          # should show 16                                                                                    
  ip addr show                   # should have an IP on enp0s2 or similar
  ping -c2 8.8.8.8               # should work                                                                                       
  virsh version                  # confirms libvirt is installed inside L1                                                           
  ls /dev/kvm                    # confirms nested KVM is exposed                                                                    
                                                                                                                                     
  Step 6 — Smoke-test save/restore (do this before trusting it)                                                                      
                                                                                                                                     
  Back on L0 (after exiting virsh console):                                                                                          
  sudo virsh save l1-ubuntu /root/nested_qemu/test.save --bypass-cache                                                             
  ls -lh /root/nested_qemu/test.save     # check file size                                                                           
  sudo virsh restore /root/nested_qemu/test.save --bypass-cache                                                                      
  sudo virsh console l1-ubuntu           # reconnect, run `uptime` — should not have rebooted                                        
  rm /root/nested_qemu/test.save                                                                                                     
                                                                                                                                     
  If this round-trip works clean, you're ready to spin up L2 VMs.

● Here are the two restore workflows we used:                                                                                         
                                                                                                                                      
  1. virsh save / virsh restore (managed save)                                                                                        
                                                                                                                                      
  Save (stops the VM, writes memory to file):                                                                                         
  sudo virsh save l1-ubuntu /root/nested_qemu/test.save --bypass-cache
                                                                                                                                      
  Restore (resumes from saved memory; disk is whatever it is now):                                                                    
  sudo virsh restore /root/nested_qemu/test.save --bypass-cache                                                                       
                                                                                                                                      
  2. External snapshot revert (manual — libvirt doesn't auto-revert externals)                                                        
                                                                                                                                      
  Create snapshot (live, disk + memory):                                                                                              
  sudo virsh snapshot-create-as l1-ubuntu \                                                                                           
    --name snap1 \                         
    --memspec file=/root/nested_qemu/snap1.mem,snapshot=external \                                                                    
    --diskspec vda,file=/root/nested_qemu/L1_system.snap1.qcow2,snapshot=external \                                                   
    --atomic                                                                                                                          
                                                                                                                                      
  Revert (3 steps — VM must be shut off first):                                                                                       
  # a. stop VM if running                                                                                                             
  sudo virsh destroy l1-ubuntu
                                                                                                                                      
  # b. reset overlay to empty (discards all writes since snapshot)                                                                    
  sudo rm /root/nested_qemu/L1_system.snap1.qcow2                                                                                     
  sudo qemu-img create -f qcow2 \                                                                                                     
    -b /root/nested_qemu/L1_system.qcow2 -F qcow2 \                                                                                   
    /root/nested_qemu/L1_system.snap1.qcow2                                                                                           
                                           
  # c. restore memory state                                                                                                           
  sudo virsh restore /root/nested_qemu/snap1.mem
                                                                                                                                      
  Key difference: virsh save/restore is a single round-trip (save stops the VM). External snapshots stay live — the VM keeps running
  after the snapshot — but reverting is manual.
